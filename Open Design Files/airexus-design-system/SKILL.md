# AirexUs - Design System

## What's inside
- `DESIGN.md`: The canonical visual rules, tokens, and anti-patterns.
- `colors_and_type.css`: Concrete CSS custom properties for the palette, spacing, and typography.
- `assets/files` and `build/files`: Preserved source branding, logos, and binary assets from the snapshots.
- `source_examples/code`: High-signal original styling configurations from the codebase.
- `preview/files`: Focused HTML cards demonstrating colors, typography, spacing, radius, shadows, and brand assets.
- `ui_kits/app/files`: A React/JSX implementation of core structural workspace UI kit components.

## Source context
This design system was extracted directly from a source B2B engineering web repository (`ccruz-bno/airexs-us-home-page` and local code `AirexsUS - Sitio Web`). The visual language relies on structural darks and high-contrast technical accents, serving as evidence for its authentic structural design.

## When to use
Activate this skill when generating prototypes, mockups, or interfaces for industrial, architectural, or HVAC engineering domains. It is specifically tuned for building production-ready B2B artifacts where clean, structured information density is required.

## How to use
1. Read `README.md` and `DESIGN.md` to understand the product domain, core rules, and anti-patterns.
2. Load `colors_and_type.css` to inject the system's root variables and typography.
3. Browse `preview/files` cards to review the palette, typography scale, and branding available in `assets/files` and `build/files`.
4. Inspect `source_examples/code` for original component implementations.
5. Check `ui_kits/app/index.html` and the component files for concrete workspace component applications.

## Design system highlights
- **Technical Accents:** Primary Yellow (`#FFE900`) and Primary Cyan (`#2B99D1`) create high-visibility calls to action over deep neutral backgrounds. These colors drive the interaction states.
- **Structural Neutrals:** Dark shades (`#21242B`, `#323641`) replace standard flat blacks to give depth to elevated layout surfaces and interactive cards.
- **Typography:** Relies exclusively on `Poppins` with balanced text wrapping for clean, modern readability.
- **Spacing and Radius:** Employs precise spacing and radius definitions to maintain a rigid structural grid.l grid.id.
